# Typeset (typography refinement)

This skill is the **typeset** workflow from the [skills.sh leaderboard #313](https://skills.sh/pbakaus/impeccable/typeset) (sourced from [pbakaus/impeccable](https://github.com/pbakaus/impeccable), installable with `npx skills add https://github.com/pbakaus/impeccable --skill typeset`). It replaces invisible defaults with intentional hierarchy: distinctive typefaces, modular scales, readable measure, and consistent weight roles.

## Mandatory preparation

Before typesetting:

1. Read **`skills/frontend-design/SKILL.md`** for aesthetic direction, font choices, and anti-patterns.
2. Read **`skills/web-design-guidelines/SKILL.md`** for contrast, readability, and accessibility guardrails.
3. If the repository uses the full **impeccable** harness (`PRODUCT.md` / `DESIGN.md`), load context per `skills/impeccable/SKILL.md`. For **brand-register** font selection, also read **`skills/impeccable/reference/brand.md`** (font procedure and reflex-reject list).

Confirm whether the surface is **brand** (fluid display scale, expressive pairing) or **product** (fixed `rem` scale, system or single-family UI type). If unclear from the codebase, ask once before swapping fonts broadly.

## Procedure

Follow **`reference/typeset.md`** end to end for register-specific assessment, type-scale planning, systematic fixes, reference material, and verification.

When hierarchy and readability are solid, continue with **`skills/polish/SKILL.md`** for the final ship pass.
