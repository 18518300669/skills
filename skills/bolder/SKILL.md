# Bolder (intentional amplification)

This skill is the **bolder** workflow from the [skills.sh leaderboard](https://skills.sh/pbakaus/impeccable/bolder) (sourced from [pbakaus/impeccable](https://github.com/pbakaus/impeccable)). It amplifies hierarchy, scale, and visual confidence without defaulting to cyan gradients, glassmorphism, or neon “AI bold.”

## Mandatory preparation

Before making a surface bolder:

1. Read **`skills/frontend-design/SKILL.md`** for distinctive type, color, and layout direction.
2. Read **`skills/web-design-guidelines/SKILL.md`** for contrast, motion, and interaction guardrails.
3. Read **`skills/impeccable/SKILL.md`** for register (brand vs product), context gathering, and anti-pattern “do not” lists.

If brand personality, audience, or how far to push the UI is unclear from the codebase, ask the user directly before inventing a dramatic direction.

## Procedure

Follow **`reference/bolder.md`** end to end for register-specific amplification, dimension-by-dimension guidance, verification, and polish handoff.

When impact is in place, continue with **`skills/polish/SKILL.md`** for the final ship pass.
