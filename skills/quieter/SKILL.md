# Quieter (visual refinement)

This skill is the **quieter** workflow from the [skills.sh leaderboard](https://skills.sh/pbakaus/impeccable/quieter) (sourced from [pbakaus/impeccable](https://github.com/pbakaus/impeccable), installable with `npx skills add https://github.com/pbakaus/impeccable --skill quieter`). It is the counterweight to **`skills/bolder/SKILL.md`**: tone down overstimulating UI without neutralizing the point of view.

## Mandatory preparation

Before making a surface quieter:

1. Read **`skills/impeccable/SKILL.md`** for register (brand vs product), context gathering, and anti-pattern lists.
2. Read **`skills/frontend-design/SKILL.md`** for aesthetic direction and craft expectations.
3. Read **`skills/web-design-guidelines/SKILL.md`** for contrast, motion, and accessibility guardrails.

Confirm the design is **too loud or aggressive**, not merely **too busy**. If the problem is too many competing elements, prefer **`skills/distill/SKILL.md`** first. If the problem is timid or generic, prefer **`skills/bolder/SKILL.md`**.

If purpose, audience, or how far to reduce intensity is unclear from the codebase, ask the user before applying broad desaturation or flattening.

## Procedure

Follow **`reference/quieter.md`** end to end for register-specific refinement, dimension-by-dimension guidance, verification, and polish handoff.

When the result feels refined but not generic, continue with **`skills/polish/SKILL.md`** for the final ship pass.
